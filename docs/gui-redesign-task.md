# AiPairLauncher GUI 重设计清单

> 目标：将当前 1061 行单窗口三栏布局拆分为**侧边导航 + 多页切换**架构，降低信息密度，提升操作效率。
> 核心原则：**首屏只展示当前任务最需要的信息，其余按需切换**。
> 2026-03-28：本文档由仓库根目录 `task.md` 迁移至 `docs/gui-redesign-task.md`，避免与运行时 `.aipair/task.md` 冲突。

---

## 一、现状问题分析

### 1.1 布局瓶颈

| 问题 | 描述 |
|------|------|
| 单窗口全塞 | 1061 行 XAML、100+ 控件挤在一个 MainWindow 里，无页面切换 |
| 三栏同时竞争 | 左栏 360px 会话列表 + 中栏弹性详情 + 右栏 430px 配置，最小宽度 1380px |
| 中栏无限滚动 | 会话概览 → 信息网格 → 元数据编辑 → 输出预览 → 上下文桥接，5 张卡片纵向堆叠 |
| 右栏手风琴爆炸 | "新建会话"面板内含 10 行配置 + 模板保存 + worktree + 启动按钮；"自动编排"面板含状态视图 + 6 项参数 + 审批流 + 阶段历史 |
| 底部日志固定 180px | 占据宝贵的垂直空间，但大部分时间用户不看日志 |
| 重复信息 | Header 区域版本号在 Footer 重复显示；会话卡片内操作按钮和中栏元数据区按钮功能重叠 |

### 1.2 用户操作流分析

典型用户场景按频率排序：

1. **启动新会话** → 填配置 → 点启动（高频，首次打开最需要）
2. **切换/监控会话** → 看状态 → 聚焦面板（高频）
3. **自动编排** → 配参数 → 审批 → 看进度（中频，需要专注）
4. **上下文桥接** → 写指令 → 发送（中频，需要专注）
5. **管理会话元数据** → 改名/分组/归档（低频）
6. **查看依赖/日志** → 诊断问题（低频）

## 二、重设计方案

### 2.1 整体架构：侧边导航 + 内容页

```
┌──────┬───────────────────────────────────────────────┐
│ Nav  │                                               │
│      │          Page Content Area                    │
│ [图] │                                               │
│ [图] │    (根据左侧导航切换不同页面)                    │
│ [图] │                                               │
│ [图] │                                               │
│ [图] │                                               │
│      │                                               │
│      ├───────────────────────────────────────────────┤
│ [设] │  Status Bar (精简：状态 + 版本)                │
└──────┴───────────────────────────────────────────────┘

Nav 宽度: 56px (图标) / 200px (展开)
```

### 2.2 页面拆分方案

将现有内容拆为 **5 个页面**：

---

#### Page 1: 仪表盘 (Dashboard) — 默认首页

> 一眼看清所有会话状态 + 快速启动入口

```
┌─────────────────────────────────────────────────┐
│  统计卡片行: [总会话数] [运行中] [等待中] [异常]    │
├─────────────────────────────────────────────────┤
│  待处理队列 (条件显示，合并现有 Row 1)              │
├───────────────────────┬─────────────────────────┤
│  会话列表 (网格/卡片)   │  快速启动面板             │
│  - 搜索 + 筛选         │  - 目录 + 工作区          │
│  - 紧凑卡片视图        │  - 模式选择              │
│  - 状态徽标            │  - 模板选择              │
│  - 单击选中 → 右侧更新  │  - [启动 Ai Pair] 按钮   │
│                       │                         │
└───────────────────────┴─────────────────────────┘
```

**设计要点：**
- 会话卡片精简：只保留名称、分组、状态徽标、最近活动时间
- 移除卡片内的操作按钮（置顶/复制/归档），统一到右键菜单或详情页
- 快速启动面板只保留核心字段，高级选项折叠
- PreviewSnippet 移到会话详情页

#### Page 2: 会话详情 (Session Detail)

> 选中某个会话后的完整信息和操作

```
┌─────────────────────────────────────────────────┐
│  Session Header: 名称 + 状态 + 操作按钮栏          │
│  [加载] [重连] [聚焦Claude] [聚焦Codex] [归档]     │
├───────────────────────┬─────────────────────────┤
│  信息面板              │  输出预览                 │
│  - 工作目录            │  ┌───────┬───────┐      │
│  - 运行时绑定          │  │Claude │Codex  │      │
│  - 模式/徽标           │  │       │       │      │
│  - 模板/worktree       ���  │       │       │      │
│                       │  └───────┴───────┘      │
├───────────────────────┴─────────────────────────┤
│  元数据编辑 (可折叠)                               │
│  - 显示名称 / 分组 / 保存                          │
```

**设计要点：**
- 从 Dashboard 双击会话卡片或点击"查看详情"跳转

- 元数据编辑默认折叠，低频操作不占空间
- 侧边栏保持会话快速切换列表（可选）

#### Page 3: 上下文桥接 (Context Bridge)

> 专注的上下文传递工作区

```
┌─────────────────────────────────────────────────┐
│  当前会话: [会话名] [状态] [快速切换下拉]           │
├─────────────────────────────────────────────────┤
│  ┌──────────────────────────────────────────┐    │
│  │  传输指令编辑区 (大面积)                     │    │
│  │                                          │    │
│  │                                          │    │
│  └──────────────────────────────────────────┘    │
├──────────────┬──────────────────────────────────┤
│  配置区       │  操作 + 状态区                     │
│  抓取行数     │  [左→右 发送]  [右→左 发送]         │
│  自动回车     │  最近成功: xxx                     │
│              │  最近失败: xxx                     │
└──────────────┴──────────────────────────────────┘
```

**设计要点：**
- 指令编辑区给予最大面积，这是核心操作
- 配置项极简化，行数和回车选项用内联控件
- 状态信息紧凑显示

#### Page 4: 自动编排 (Automation)

> 自动编排的完整工作台

```
┌─────────────────────────────────────────────────┐
│  当前会话: [会话名] [Phase徽标] [启动] [停止]       │
├───────────────────────┬─────────────────────────┤
│  编排状态              │  审批面板                 │
│  - Phase 进度条/步骤   │  - 当前待审批阶段         │
│  - task.md 状态        │  - 摘要                  │
│  - 任务进度            │  - 审批备注输入           │
│  - 状态详情文本        │  - [批准] [退回]          │
├───────────────────────┼─────────────────────────┤
│  任务提示词 (编辑区)    │  阶段历史 (时间线视图)     │
│                       │  - 按时间排列             │
│                       │  - 状态 + 摘要            │
├───────────────────────┴─────────────────────────┤
│  高级参数 (可折叠)                                 │
│  [推进策略] [轮询ms] [超时s] [行数] [阶段数] [重试] │
└─────────────────────────────────────────────────┘
```

**设计要点：**
- Phase 进度用步骤条可视化（Phase 1→2→3→4），比纯文本直观
- 6 项高级参数默认折叠，大部分用户用默认值
- 审批区域突出显示，有待审批时自动展开
- 阶段历史换成竖向时间线，比 ListBox 更清晰

**设计要点：**
- 主题切换从 Header 移到设置页

- 依赖检查从右栏 Expander 移入
- 缓存清理、孤儿 worktree 清理等维护操作归入

### 2.3 系统日志处理

**方案：日志面板改为浮动抽屉 (Drawer)**

- 底部固定一个 32px 的状态栏，显示最新一条日志摘要
- 点击状态栏或快捷键 `Ctrl+L` 展开日志抽屉（从底部滑出，覆盖内容区 40%高度）
- 再次点击或按 Esc 收起
- 释放 180px 固定高度给内容区

### 2.4 Header / Footer 精简

**现有问题：**
- Header: 标题 + 副标题 + 版本 + 会话计数 + 待处理计数 + 主题选择器 → 占 ~80px
- Footer: 状态消息 + 版本（与 Header 重复）→ 占 ~50px
- 合计 ~130px 被非核心信息占用

**改为：**
- Header 移除，标题融入侧边导航顶部 logo
- 统计徽标融入 Dashboard 页面顶部
- Footer 改为 32px 状态栏：`[最新日志摘要...] [v1.x.x]`
- 主题切换移入设置页
- **净释放 ~100px 垂直空间**

## 五、风险与约束

1. **数据绑定迁移** — 现有 ViewModel 高度耦合，拆分时需注意属性依赖链
2. **事件处理器迁移** — MainWindow.xaml.cs 有 1230 行事件处理代码，需逐一迁移到对应页面
3. **页面间状态同步** — 会话选中状态需要在 Dashboard 和 SessionDetail 间同步
4. **自动编排实时更新** — 切换到其他页面时，自动编排状态仍需后台更新，回到页面时需立即反映
5. **��后兼容** — 用户已习惯的操作路径（如从会话列表直接看到输出预览）需提供替代方案
