# AiPairLauncher 下一步开发清单

> 聚焦方向：**稳定性加固** + **UI/UX 体验优化**
> 基准版本：v1.2.2 (2026-03-29)
> 前置完成：GUI 重设计 Phase 1-3 已落地（侧边导航 + 5 页架构）

---

## 一、稳定性加固

### 1.1 轮询与容错机制

- [ ] **S1** 会话监控引入断路器模式
  - SessionMonitorService 8s 轮询无退避，WezTerm 无响应时持续轰炸
  - 方案：连续 N 次超时后指数退避（8s → 16s → 32s → 上限 60s），恢复后回到 8s
  - 在 UI 上显示"终端无响应，已降低检测频率"提示

- [ ] **S2** 自动编排心跳超时检测
  - AutoCollaborationCoordinator 长时间无输出变化时无超时机制，可能卡死
  - 方案：增加可配置的阶段超时（默认 10 分钟），超时后暂停并通知用户决策
  - 提供"继续等待 / 重试本阶段 / 终止编排"三选一

- [ ] **S3** ProcessRunner 资源清理加固
  - 进程超时后 TryKillProcess 吞异常，极端情况下僵尸进程残留
  - 方案：kill 后等待退出确认（WaitForExit 500ms），失败则记录警告日志
  - 补充 IDisposable 模式确保 Process 对象释放

### 1.2 数据持久化

- [ ] **S4** SQLite 数据库 Schema 版本管理
  - SessionStore 无 Schema 版本号，升级时结构变更可能导致数据丢失
  - 方案：增加 `schema_version` 表，启动时检查版本并执行增量迁移脚本
  - 迁移失败时自动备份旧库文件，提示用户

- [ ] **S5** 自动编排快照与 Worktree 一致性校验
  - 自动编排快照引用的 worktree 路径可能已被手动删除
  - 方案：恢复快照时校验 worktree 是否存在，不存在则标记为"快照失效"并提示

### 1.3 异常处理与可观测性

- [ ] **S6** WezTerm 通信失败分级处理
  - 当前 catch-all 统一转为"已断开"状态，无法区分网络超时 / 进程退出 / 权限拒绝
  - 方案：按异常类型分级（Timeout → 重试提示、ProcessExit → 重启提示、其他 → 详情日志）
  - 状态栏显示具体断开原因

- [ ] **S7** 崩溃日志增加上下文信息
  - 当前 crash log 只记录异常堆栈，缺少会话状态、自动编排阶段等上下文
  - 方案：crash log 附加当前活动会话 ID、编排 Phase、最近 5 条操作日志

- [ ] **S8** IME 输入崩溃防御常态化
  - SessionDetailPage 已两次修复 IME 崩溃，说明 WPF TextBox + 中文输入法交互脆弱
  - 方案：封装 SafeTextBox 控件统一处理 IME 组合事件异常，全局替换高风险输入框

### 1.4 会话生命周期

- [ ] **S9** 僵尸会话自动检测与清理
  - 会话无限持久化，WezTerm 进程退出后会话仍显示"运行中"直到下次刷新
  - 方案：轮询检测到进程不存在时自动标记为"已断开"，超过 24h 未重连的提示归档

- [ ] **S10** WezTerm 崩溃后的自动恢复引导
  - WezTerm 意外退出后用户需手动"刷新 → 重连"，流程不直观
  - 方案：检测到 WezTerm 进程退出时弹出恢复向导："重启终端并恢复会话 / 归档会话"

## 二、UI/UX 体验优化

### 2.1 首次使用体验

- [ ] **U1** 新手引导流程
  - 首次启动时检测依赖状态，缺失时展示分步安装引导而非静默错误
  - 引导步骤：检测 WezTerm → 检测 Claude CLI → 检测 Codex CLI → 完成
  - 每步提供一键安装/跳转链接

- [ ] **U2** 快速启动面板优化
  - 新用户面对"目录 + 工作区 + 分组 + 模式 + 模板"字段不知如何开始
  - 方案：默认只显示"工作目录"一个必填项 + "启动"按钮，其余折叠到"更多选项"
  - 提供"最近使用的目录"下拉快速选择

### 2.2 会话管理体验

- [ ] **U3** 会话状态实时反馈增强
  - 当前状态变更（启动中 → 运行中 → 断开）无过渡动画，用户难以感知
  - 方案：状态切换时卡片短暂高亮 + 状态徽标渐变动画
  - 运行中的会话显示呼吸灯效果

- [ ] **U4** 会话列表分组与排序
  - 会话多时（10+）列表拉长，查找困难
  - 方案：支持按分组折叠/展开、按状态筛选、按最近活动排序
  - 置顶会话始终显示在顶部

- [ ] **U5** 批量会话操作
  - 多会话场景下逐个归档/清理效率低
  - 方案：会话卡片增加复选模式，支持批量归档、批量清理

### 2.3 自动编排体验

- [ ] **U6** Phase 进度可视化升级
  - 当前 Phase 进度为文本描述，不够直观
  - 方案：Phase 1→2→3→4 步骤条组件，当前阶段高亮 + 已完成打勾 + 未到达置灰
  - 每个 Phase 下方显示预估/已用时间

- [ ] **U7** 自动编排启动前置条件检查
  - 已有两次提交修复"启动条件不清晰"问题，说明用户仍然困惑
  - 方案：启动按钮旁增加前置条件清单（会话运行中 ✓、task.md 存在 ✓、Phase 配置完成 ✓）
  - 条件不满足时按钮置灰并 tooltip 说明原因

- [ ] **U8** 审批通知增强
  - 用户不在 Automation 页面时可能错过审批请求
  - 方案：待审批时侧边导航图标显示红点 + Windows 系统托盘通知（可选）
  - 点击通知直接跳转审批面板

### 2.4 输出预览与上下文桥接

- [ ] **U9** 输出预览自动滚动与搜索
  - 当前输出预览为静态文本块，长输出时难以定位关键信息
  - 方案：增加自动滚动到底部开关 + Ctrl+F 文本搜索 + 关键词高亮

- [ ] **U10** 上下文桥接历史记录
  - 发送过的指令无历史，重复发送需重新输入
  - 方案：保留最近 20 条发送记录，支持一键重发

### 2.5 响应式与可访问性

- [ ] **U11** 降低最小窗口宽度要求
  - 当前 1380px 最小宽度对笔记本用户不友好（1366px 屏幕无法全屏使用）
  - 方案：目标降至 1100px，Dashboard 在窄屏时快速启动面板移至会话列表下方
  - gui-redesign-task.md T4.4 遗留项

- [ ] **U12** 全局快捷键支持
  - gui-redesign-task.md T3.4 遗留项
  - `Ctrl+N` 新建会话、`Ctrl+L` 日志抽屉、`Ctrl+1~5` 页面切换
  - 设置页展示快捷键速查表

- [ ] **U13** 主题跟随系统深色模式
  - 当前三套主题需手动切换
  - 方案：增加"跟随系统"选项，监听 Windows 深色模式切换事件自动适配

## 五、实施建议

1. **按 P0 → P1 → P2 顺序推进**，每个任务独立分支 `fix/<编号>` 或 `feat/<编号>`
2. **S1 + S6 + S9 可合并为一个"会话健康监控重构"分支**，减少 SessionMonitorService 反复改动
3. **U1 + U2 合并为"首次使用体验优化"**，一次性改善新用户上手流程
4. **每个 P0 任务完成后跑全量测试**，确认无回归
5. **P2 任务可作为社区贡献友好项**，issue 标注 `good-first-issue`
